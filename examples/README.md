## 簡報時建議的 Demo 順序

### 🎯 推薦組合 A：5 分鐘短 Demo（時間有限）

只用 1 份範例：**`business_idea_solo_freelancer_app.md`**

> 「假設我是一位自由工作者，想推出一款叫 SoloOps 的 App。我把企劃書丟進 MiroFish ——」
>
> *（拖檔案進去）*
>
> 「五分鐘後，我會拿到一份報告告訴我這個點子會紅、不會紅、還可以改哪裡。」

### 🚀 推薦組合 B：10 分鐘完整 Demo（標準長度）

順序：**SoloOps → 一碗 → SilentMatch**

1. 用 SoloOps 展示「典型 SaaS 點子被驗證」
2. 用「一碗」展示「不只科技業，餐飲業也能用」
3. 用 SilentMatch 展示「**MiroFish 也能告訴你不會紅**」← 這個會讓觀眾印象最深

## 在簡報中強調的 Demo 點

### ⭐ Demo Point 1：上傳後的「即時理解」
打開後立刻可以看到 MiroFish 自動萃取：
- 核心賣點是什麼
- 目標客群
- 潛在風險
- 競品定位

### ⭐ Demo Point 2：虛擬市場的「人格多樣性」
讓觀眾看到生成的虛擬消費者**有不同年齡、職業、消費習慣**——不是 AI 隨便生成的同一個人。

### ⭐ Demo Point 3：報告的「為什麼」
不只是「成功機率 65%」這種數字，而是：

> 「**為什麼會紅**：它解決了自由工作者最痛的『追款問題』。
> **為什麼可能不會紅**：NT$299 對自由工作者來說已經是一杯星巴克 + 一頓便當的價格，會猶豫。
> **建議改進**：可以推出 14 天免費試用 + 第一個月 NT$99 的入門優惠。」

### ⭐ Demo Point 4：跟虛擬消費者「直接對話」
從 Step 5 點進任意一位虛擬消費者，問：

> 「你會用 SoloOps 嗎？為什麼？」

讓觀眾看到 AI 用戶會回出**有個性、有理由**的拒絕／接受答案，而不是罐頭回覆。

## 自己想做新範例？

照著這三份的格式寫即可。**好範例的關鍵元素**：

1. **明確的目標客群**（年齡、職業、地點）
2. **具體的價格**（讓 Agent 有東西可以討論）
3. **明確的痛點**（為什麼有人會買單）
4. **誠實的風險**（讓 MiroFish 能找出可改進處）
5. **清楚的「想驗證什麼」**（給系統明確指引）

格式可以是 `.md`、`.txt`、`.pdf`，MiroFish 都支援。
